# Red Team Command Doctrine

## Purpose
Use this skill when red-team mode is active and the task needs offensive discipline without heavy prompt injection.

## Use pattern
1. Identify the current phase.
2. Select the most concrete router first.
3. Prefer the matching detailed pack before abstract methodology if the technical path is already visible.
4. Route into the most specific leaf skill.
5. Open only the matching references needed for the current path.
6. If the task is large or multi-step, switch to structured orchestration instead of free-form planning.
7. End with one concrete next step.
