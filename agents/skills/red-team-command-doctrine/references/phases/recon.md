# Recon

## Objective
Map the reachable surface and isolate one promising path.

## Enter when
You do not yet have execution, valid credentials, or a proven exploit chain.

## Prioritize evidence
- exposed hosts/routes
- auth boundaries
- tech stack fingerprints
- hidden endpoints/parameters

## Low-noise path first
Prefer one focused map of the likely path over broad scanning.

## Typical skills/tools
- `recon-for-sec`, `recon-and-methodology`
- web-access, Burp, targeted CLI recon

## Exit when
You can name one concrete initial-access or exploit hypothesis worth proving next.
