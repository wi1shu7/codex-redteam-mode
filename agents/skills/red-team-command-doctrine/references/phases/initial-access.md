# Initial Access

## Objective
Gain first controlled execution, session, or authenticated foothold.

## Enter when
Recon has isolated a viable entry path.

## Prioritize evidence
- exploitability of the path
- auth/session state
- target-side validation behavior
- delivery constraints

## Low-noise path first
Prove the smallest viable access path before trying multiple delivery formats.

## Typical skills/tools
- `initial-access-delivery`
- exploit-specific web/auth skills

## Exit when
You have controlled code execution, a stable session, or valid target access.
